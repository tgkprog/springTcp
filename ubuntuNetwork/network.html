RELAY SERVER Java 26 Pipe Relay Design

We plan to build a clean, bidirectional TCP forwarder in `/data/code/gt/tcp_tgk/relayServer/RelayServer.java` compiled and run directly using Java 26.

Using Java 26's native Virtual Threads (Project Loom) allows us to spawn dedicated connection-handling worker loops without thrashing resources or needing complex non-blocking NIO abstractions.

Planned Class Architecture

RelayServer.java Outline 
import java.io.*;
import java.net.*;

public class RelayServer {
    private static final int BUFFER_SIZE = 8192;

    public static void main(String[] args) {
        if (args.length < 3) {
            System.err.println("Usage: java RelayServer <listen-port> <vps-ip> <vps-port>");
            System.exit(1);
        }
        int listenPort = Integer.parseInt(args[0]);
        String vpsHost = args[1];
        int vpsPort = Integer.parseInt(args[2]);

        System.out.printf("Relay Server starting: listening on port %d, forwarding to %s:%d%n", 
            listenPort, vpsHost, vpsPort);

        try (ServerSocket serverSocket = new ServerSocket(listenPort)) {
            while (true) {
                Socket clientSocket = serverSocket.accept();
                // Spawn a virtual thread to handle each incoming client session
                Thread.startVirtualThread(() -> handleConnection(clientSocket, vpsHost, vpsPort));
            }
        } catch (IOException e) {
            System.err.println("Server exception: " + e.getMessage());
        }
    }

    private static void handleConnection(Socket clientSocket, String vpsHost, int vpsPort) {
        System.out.printf("New client connection from %s%n", clientSocket.getRemoteSocketAddress());
        try (Socket vpsSocket = new Socket(vpsHost, vpsPort)) {
            // Configure TCP socket properties (matching client/server low latency setup)
            clientSocket.setTcpNoDelay(true);
            vpsSocket.setTcpNoDelay(true);

            // Create two virtual threads for bidirectional data copy
            Thread clientToVps = Thread.startVirtualThread(() -> pipe(clientSocket, vpsSocket, "Client -> VPS"));
            Thread vpsToClient = Thread.startVirtualThread(() -> pipe(vpsSocket, clientSocket, "VPS -> Client"));

            // Wait for both directions to finish
            clientToVps.join();
            vpsToClient.join();
        } catch (IOException | InterruptedException e) {
            System.err.println("Connection error: " + e.getMessage());
        } finally {
            closeQuietly(clientSocket);
            System.out.println("Session closed cleanly.");
        }
    }

    private static void pipe(Socket source, Socket dest, String direction) {
        try (InputStream in = source.getInputStream();
             OutputStream out = dest.getOutputStream()) {
            byte[] buffer = new byte[BUFFER_SIZE];
            int read;
            while ((read = in.read(buffer)) != -1) {
                out.write(buffer, 0, read);
                out.flush();
            }
        } catch (IOException e) {
            // Socket closed or network error is expected during fault simulation
            System.out.printf("[%s] Pipe ended: %s%n", direction, e.getMessage());
        } finally {
            // Close connection when one end finishes
            closeQuietly(dest);
        }
    }

    private static void closeQuietly(Socket socket) {
        try {
            if (socket != null && !socket.isClosed()) {
                socket.close();
            }
        } catch (IOException ignored) {}
    }
}

PLAN Verification steps
  1. 1. Setup Namespaces
    Run the Setup script to build the virtual networking topology.
  2. 2. Verify Routes
    Ping `10.200.2.2` from `netns_client` to confirm routing.
  3. 3. Boot Relay
    Start Java 26 Relay inside `netns_relay` bound to `10.200.2.2:X1`.
  4. 4. Launch Client
    Start the GUI client inside `netns_client` and connect to `10.200.2.2:X1`.
  5. 5. Induce Faults
    Disable interface `veth_c` on host. Confirm client disconnect logs and automatic retry state.
  6. 6. Restore
    Bring interface `veth_c` back up. Confirm client auto-recovers and reconnects via the Relay.
