CONNECTION MONITORING FEATURE - SUMMARY
================================================================================

✅ IMPLEMENTED: Optional connection health monitoring for TCP client

FEATURES:
---------
1. Atomic Status Tracking
   - States: UNKNOWN, CONNECTING, CONNECTED, ERROR
   - Thread-safe via AtomicReference
   - Accessible via client 'status' command

2. Scheduled Health Checks
   - Sends probe:shallow periodically
   - Default interval: 2000ms (configurable)
   - Disabled by default (opt-in feature)

3. Automatic Error Recovery
   - ERROR state is transient
   - Immediately triggers reconnection
   - Background thread for non-blocking reconnect
   - Flow: ERROR → CONNECTING → CONNECTED

4. Configuration
   tcp.client.monitoring.enabled: false (default)
   tcp.client.monitoring.tick-ms: 2000 (default)

USAGE:
------
Enable in application.yml:
  tcp:
    client:
      monitoring:
        enabled: true
        tick-ms: 2000

Or via command-line:
  java -jar tcp-client.jar \
    --tcp.client.monitoring.enabled=true \
    --tcp.client.monitoring.tick-ms=3000

Check status in client:
  > status
  Connection Status: CONNECTED
